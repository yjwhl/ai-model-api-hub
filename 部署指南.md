# 🚀 GitHub 部署指南

## 准备工作

### 1. 安装 Git
下载安装 Git: https://git-scm.com/download/win

### 2. 创建 GitHub 仓库
1. 登录 GitHub: https://github.com
2. 点击右上角 `+` → `New repository`
3. 仓库名称：`ai-model-api-hub`（或其他你喜欢的名字）
4. 选择 `Public`
5. 点击 `Create repository`

### 3. 初始化本地仓库并推送

在项目文件夹打开终端（PowerShell 或 Git Bash），执行：

```bash
# 初始化
git init

# 添加所有文件
git add .

# 提交
git commit -m "feat: 大模型API汇总网站 v1.0"

# 添加远程仓库（替换 YOUR_USERNAME 和 YOUR_REPO 为你的用户名和仓库名）
git remote add origin https://github.com/YOUR_USERNAME/YOUR_REPO.git

# 推送
git branch -M main
git push -u origin main
```

## 启用 GitHub Pages

1. 进入你的仓库 → `Settings`
2. 左侧菜单找到 `Pages`
3. Source 选择 `Deploy from a branch`
4. Branch 选择 `main` 和 `/(root)`
5. 点击 `Save`
6. 等待 1-2 分钟，网站即可访问

访问地址：`https://YOUR_USERNAME.github.io/YOUR_REPO/`

## 自动化数据更新

### 启用 GitHub Actions

1. 推送代码后，GitHub Actions 会自动运行
2. 进入仓库 → `Actions` 查看部署状态
3. 首次部署需要手动确认 Pages 访问权限

### 手动触发更新

1. 进入仓库 → `Actions`
2. 选择 `Auto Update Model Data`
3. 点击 `Run workflow` → `Run workflow`

## 自定义域名（可选）

1. 购买域名（如阿里云、腾讯云）
2. 在 GitHub Pages 设置中添加自定义域名
3. 配置 DNS 解析指向 GitHub

## 项目文件结构

```
ai-model-api-hub/
├── index.html          # 主页面
├── README.md           # 项目说明
├── .gitignore          # Git忽略文件
├── .github/
│   └── workflows/
│       └── update-models.yml  # 自动更新工作流
└── js/
    └── models.js       # 模型数据（独立文件，方便更新）
```

## 数据更新流程

### 手动更新
直接编辑 `js/models.js` 文件中的 `MODELS_DATA.models` 数组

### 数据格式
```javascript
{
    id: "unique-id",
    name: "模型名称",
    provider: "厂商名称",
    inputPrice: 0.00,    // 美元/千Token
    outputPrice: 0.00,
    contextWindow: "128K",
    features: ["功能1", "功能2"],
    specialties: "核心优势描述",
    badges: ["new", "hot", "free", "opensource"],
    url: "官方API地址"
}
```

### 自动更新（高级）
可以接入第三方 API 或使用爬虫自动抓取最新价格数据，修改 `.github/workflows/update-models.yml` 中的脚本逻辑。

## ❓ 常见问题

**Q: GitHub Pages 显示 404？**
A: 等待几分钟后刷新，或检查 Settings → Pages 配置

**Q: 推送代码需要输入密码？**
A: 使用 Personal Access Token 代替密码，或配置 SSH Key

**Q: 如何让更多人看到？**
A: 发布到 V2EX、掘金、知乎等平台，分享项目链接
